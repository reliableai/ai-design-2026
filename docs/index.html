Course Description

This course provides a comprehensive introduction to designing, building, and evaluating large-scale AI systems. Students will gain hands-on experience building AI-powered applications and learn rigorous evaluation methodologies essential for deploying AI systems in production.

Learning Objectives

By the end of this course, you will be able to:

  • Design and implement AI-powered applications using agents, tools, and memory systems
  • Understand and apply MCP-style connectors for system integration
  • Design comprehensive evaluation frameworks for AI systems
  • Create and refine golden datasets for testing
  • Implement LLM-as-judge evaluation approaches
  • Conduct rigorous experimental design and ablation studies
  • Perform error analysis and quantify uncertainty in AI system outputs
  • Architect scalable AI systems with appropriate safety guardrails
  • Communicate technical design decisions and evaluation results effectively

Course Structure

The course spans 12 weeks with two 90-minute lessons per week (24 lessons total). It is organized into three phases:

Phase 1: Building & Evaluating AI Apps (Lessons 1-6)

Hands-on introduction to building and evaluating AI applications. Covers LLM API calls, the agentic loop, context and memory, monitoring and observability, quality estimation, and systematic iteration.

Phase 2: Complex AI Systems (Lessons 7-10)

Scaling up to multi-agent architectures: councils of agents, building complex systems, middleware and protocols (MCP), guardrails, privacy, and project structuring.

Phase 3: Project-Driven Work (Lessons 11-24)

Apply everything in a substantial project. Includes architecture deep dive, project specification, midterm exam, reviews, working sessions, and final presentations.

Course Policies

Attendance

Attendance is expected for all lessons. Phase 3 sessions are particularly important as they include project reviews, working sessions, and presentations.

Academic Integrity

All work submitted must be your own (or your team's for the project). You may use AI tools to assist with coding and learning, but you must understand and be able to explain all code and concepts in your submissions.

Late Submissions

Late submissions for the project milestones will be penalized unless prior arrangements are made. The midterm exam must be taken at the scheduled time.

Collaboration

The midterm exam is strictly individual. For the project, teams of up to 2 students are allowed. All team members are expected to contribute equally.

Office Hours

Office hours are held right after each class. Additional appointments can be scheduled via email.
